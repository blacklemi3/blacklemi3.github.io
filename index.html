Robot 双 MCU 机器人控制链路

一台由 ESP32-S3 主控和 Hi3863 运动控制板协作的机器人系统。主控侧负责语音、联网、推理、显示和指令调度；运动侧负责 UART 协议解析、PCA9685 舵机控制和动作执行。

  • 语音唤醒、ASR、LLM、TTS、双眼屏幕、飞书远程通道和 Flutter App 形成完整交互链路。
  • 运动控制包含帧头、CRC、ACK/NACK、自动重试、动作队列和 8 路舵机控制。
  • 公开版本只会保留协议和控制思路，不公开密钥、PCB、3D 文件、真实固件和内部交接包。

Drivers 嵌入式外设驱动与试验箱案例

围绕 GPIO、I2C、PWM、UART、ADC、OLED、PCA9685、数码管、传感器、灯带、电机和语音模块形成的一组外设驱动与场景案例。

  • Python 驱动包含 OLED 帧缓冲、PCA9685 舵机、AHT20、TM1637/TM1650、WS2812、语音识别、红外、超声波等模块。
  • C 侧案例覆盖智能场景设备、试验箱房间模块和基础物联网控制逻辑。
  • 后续公开仓库会把真实驱动抽象成通用接口和 mock 测试，而不是发布原始课程包。

ArkTS 喝啥：消费级轻产品 App

一个面向年轻消费场景的 HarmonyOS NEXT / ArkTS App，用抽卡方式解决“今天喝什么”的选择困难，并通过签到、成就、图鉴、等级和金币形成持续使用路径。

  • 需求文档明确 18-35 岁用户、3 秒内给答案、打开到结果不超过 5 秒。
  • 工程按 MVVM 分层组织页面、组件、服务、模型、存储和测试。
  • 规则系统包含概率、稀有度、保底、时间权重、历史权重、分享次数和本地持久化。

Network 网络变更与自动回滚

在现有网络硬件基础上通过 VLAN Trunk 拆分办公网和业务网，并把远程变更做成带备份、自动回滚、取消回滚和验证日志的操作流程。

  • 变更前自动备份 network、dhcp、firewall 配置。
  • 内置 5 分钟自动回滚，远程 SSH 失联时能够恢复原配置。
  • 交付记录保留架构选择、实施步骤、验证结果和风险边界。
