脱敏交付记录

从资料、网站到可接手系统

这不是某一个源码仓库，而是一类真实交付经验的公开整理：在资料分散、需求变化、上线和后续维护都有约束的情况下，把页面、后台、数据、搜索可见性和交接文档收敛成可继续运行的系统。

工作内容

三条交付线

Web / 后台

门户与后台内容系统

面向门户类网站，处理前台页面、栏目结构、后台管理和内容发布流程，让非技术使用者可以持续维护内容。

  • 前台页面结构和栏目内容组织
  • 后台录入、列表、发布和维护流程
  • 基础部署、问题排查和交付衔接
Data / 复核

资料查询与方案复核

把分散资料整理成可查询、可校验、可复核的数据流程，区分原始数据、结构化结果、人工备注和最终报告。

  • 资料字段、目录结构和缓存文件设计
  • 多来源交叉检查和异常项标记
  • 规则化分档、风险提示和 Markdown 报告
Site / 可见性

官网与搜索可见性

从零搭建静态站点，并补齐结构化数据、站点地图、图片优化、搜索验证、Nginx 配置和备案迁移准备。

  • 多页面静态站、服务内容和案例结构
  • sitemap、OG 图、WebP 图片和结构化数据
  • 部署配置、搜索验证和后续维护清单

方法

交付流程

  1. 盘点资料先把页面、文档、数据、素材、账号事项和待确认问题分层，明确哪些能公开、哪些需要内部保留。
  2. 抽出结构把零散内容拆成栏目、字段、脚本、配置、报告和交接项，减少“靠人记住”的部分。
  3. 做可验证结果每个结果都尽量留下可复跑命令、样例输出、截图、校验表或报告，而不是只给口头结论。
  4. 补齐上线细节处理静态资源、路径、搜索验证、Nginx、安全头、备案准备和迁移事项，避免只完成页面但无法接手。
  5. 写交接材料把运行方式、维护步骤、风险点、后续工作和不公开内容写清楚，让后续人员知道从哪里继续。

产出

留下来的东西

可运行工具查询脚本、本地工作台、静态站构建辅助脚本、报告生成流程。
可复核数据字段说明、样例数据、异常清单、交叉验证记录和人工检查项。
可维护页面门户页面、内容栏目、官网页面、图片资源、sitemap 和搜索验证文件。
可交接文档README、部署说明、备案准备清单、后续维护建议和公开边界说明。

公开版本

对应公开材料

admission-data-workbench 用虚构样例数据演示资料查询、分档和复核报告生成。 local-handover-workbench 用虚构样例工作区演示项目扫描、运行检查和交接说明生成。 local-project-indexer 扫描本地项目结构，整理技术栈、文档和风险线索。

公开边界

没有公开的内容

保留方法

公开页面只保留方法、流程、产出类型和可运行的虚构样例项目。

去除细节

不公开客户名称、真实域名、后台地址、账号、联系方式、合同报价、学生资料、部署凭据和内部交付包。
